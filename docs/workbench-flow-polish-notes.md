# Workbench Flow Polish Notes

## 背景

这一轮优化不改动整体框架，仍然保留以下主线：

- 识别
- 建议
- 话后总结
- 下一步

同时继续保留 Agent 协同思路，不把页面做成单点问答页，也不把右侧做成重执行后台。

## 这轮重点

这一轮不是新增功能，而是优化下面三个问题：

1. 流程是否能讲清楚、跑通顺
2. 座席是否能快速扫到当前重点
3. 会中识别、会后收口、下一步处置之间是否是一套口径

## 主要判断

### 1. 顶栏要承担流程总览，不只是操作入口

当前页面最容易出现的问题，是用户能看到模块，但不一定知道“现在系统处理到哪一步了”。

所以顶栏需要补足三类信息：

- 当前处理状态
- 当前 Agent 阶段
- 当前轮次进度

这样在演示时，不需要额外解释“为什么这里开始出现会后内容”。

### 2. 会话流必须帮助用户追踪当前轮次

自动推进模式下，如果当前轮次不够醒目，用户需要自己在时间轴里找位置，会打断演示节奏。

因此会话流应做到：

- 明确标出当前轮次
- 显示已推进到第几轮
- 自动把视图跟到当前轮次附近

这样用户看页面时，注意力可以稳定停留在“当前发生了什么”。

### 3. 中间主卡需要告诉座席“现在该看什么”

“会话理解与处置”是整页的核心价值区，但如果只展示字段，不提示阶段重点，用户仍然需要自己判断：

- 当前是在会中识别阶段
- 还是已经进入会后收口阶段
- 当前应该优先看摘要、建议还是风险点

所以主卡里需要有阶段提示，让页面主动告诉座席当前阅读重点。

### 4. 右侧建议区需要和中间卡说同一种语言

右侧如果只展示动作字段，容易像另一个孤立模块。

因此右侧建议区需要明确回答两件事：

- 当前进入了哪条处理路径
- 当前建议的执行方式是什么

这样它就不再像额外的操作面板，而是会后收口的自然延续。

## 这轮优化结果

### 顶栏

- 补充 Agent 当前阶段
- 补充会话轮次进度
- 补充一句流程提示文案

### 会话流

- 增加已推进轮次提示
- 自动滚动到当前轮次

### 会话理解与处置

- 增加阶段提示条
- 会中与会后的阅读重点更明确
- 会中折叠标题直接带出当前提示状态

### 下一步处置建议

- 明确展示当前处理路径
- 明确展示当前执行方式
- 保持可编辑，但整体仍然是建议区而不是执行控制台

## 为什么这轮更适合座席

相较之前，这一轮更强调：

- 少解释：页面自己说明当前阶段
- 少跳读：当前轮次自动跟随
- 少对照：中间区和右侧区状态口径一致
- 少二次总结：系统把“现在该做什么”直接告诉座席

## 没有做的事

这一轮刻意没有做下面这些内容：

- 不新增后端
- 不接真实大模型
- 不增加新的复杂工作台区域
- 不把右侧改成强执行流转台
- 不做全局重构

## 后续建议

如果要继续往 v1.0 收口，建议优先验证：

1. 自动推进和逐轮推进下，三类场景是否都能顺着讲完
2. 座席是否能在 10 秒内看明白当前阶段和下一步建议
3. 会中强提醒场景下，中间卡和右侧建议是否仍然保持同一口径
4. 会后提交后，沉淀结果是否足够“弱提示但可感知”

如果要继续往 v2.0 走，可以在这一版基础上接：

- Agent 编排页
- 端云协同扩展字段
- 更细的任务流转状态
- 真实接口和后端编排
